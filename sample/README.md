# Sample Events

전체 Kaggle 데이터는 1억 row가 넘는다. 처음부터 전체 데이터를 Spark로 돌리면 디버깅이 어렵다.

그래서 이 디렉토리에는 세션화 규칙과 KST partition을 검증하기 위한 작은 수작업 샘플을 둔다.

## 파일

```text
sample/sample_events.csv
```

## 샘플이 검증하는 것

이 샘플은 단순히 row 수를 줄인 데이터가 아니다. 과제의 핵심 조건을 일부러 담은 테스트 데이터다.

검증하려는 케이스는 다음과 같다.

```text
1. 같은 user_id에서 5분 미만 간격이면 같은 세션
2. 같은 user_id에서 정확히 5분 간격이면 새 세션
3. 같은 user_id에서 5분 초과 간격이면 새 세션
4. 서로 다른 user_id는 독립적으로 세션화
5. UTC timestamp가 KST 기준 다음 날짜로 넘어가는 경우
6. brand가 비어 있는 nullable field 케이스
```

## 세션 경계 조건

새 세션이 시작되는 조건은 다음과 같다.

```text
previous event is null
OR current event_time - previous event_time >= 300 seconds
```

요구사항이 “5분 이상”이므로 `299초`는 같은 세션이고, `300초`는 새 세션이다.

## 기대 결과

| user_id | event_time UTC | KST dt | 이전 이벤트와 간격 | expected session_seq | 설명 |
|---:|---|---|---:|---:|---|
| 100 | 2019-10-01 00:00:00 UTC | 2019-10-01 | null | 1 | user 100의 첫 이벤트 |
| 100 | 2019-10-01 00:04:59 UTC | 2019-10-01 | 299초 | 1 | 5분 미만이므로 같은 세션 |
| 100 | 2019-10-01 00:09:59 UTC | 2019-10-01 | 300초 | 2 | 정확히 5분이므로 새 세션 |
| 100 | 2019-10-01 00:15:00 UTC | 2019-10-01 | 301초 | 3 | 5분 초과이므로 새 세션 |
| 200 | 2019-10-01 00:01:00 UTC | 2019-10-01 | null | 1 | user 200의 첫 이벤트 |
| 200 | 2019-10-01 00:03:00 UTC | 2019-10-01 | 120초 | 1 | 같은 세션 |
| 300 | 2019-10-01 15:30:00 UTC | 2019-10-02 | null | 1 | KST 기준 다음 날짜 |
| 300 | 2019-10-01 15:34:59 UTC | 2019-10-02 | 299초 | 1 | 같은 세션 |
| 300 | 2019-10-01 15:39:59 UTC | 2019-10-02 | 300초 | 2 | 새 세션 |

## KST 변환 확인

다음 timestamp는 UTC 기준으로는 10월 1일이지만, KST 기준으로는 10월 2일이다.

```text
2019-10-01 15:30:00 UTC
= 2019-10-02 00:30:00 KST
```

따라서 `user_id=300`의 row들은 다음 partition에 저장되어야 한다.

```text
dt=2019-10-02
```

## 기대 집계

샘플 전체 기준 기대값은 다음과 같다.

```text
input rows = 9
distinct user_id = 3
distinct generated sessions = 6
KST partitions = 2019-10-01, 2019-10-02
```

partition별 기대값:

| dt | rows | distinct users | distinct generated sessions |
|---|---:|---:|---:|
| 2019-10-01 | 6 | 2 | 4 |
| 2019-10-02 | 3 | 1 | 2 |

이 샘플이 통과하면 전체 데이터 처리 전에 핵심 로직을 상당 부분 검증할 수 있다.
